半路出家的前端工程師面試 - 揪竟什麼是 Senior ?

發表於 2018-08-16 | 分類於 Note


擔任前端工程師一年多，吸收了日月精華後，為了回顧這一年的經驗值成長，我本來就預計要來個巡迴面試讓市場估一下現在自己的身價如何，來看看是不是當一個網路工程師就一定要有「本科出身」的天命才能勝任，於是最近就默默開始莫名的請假 … ⁉️

去年轉職的面試心得是寫在 Logdown，當時面試了大概十家左右，抱著初出茅廬的羔羊姿態到處被洗臉，原本在製造業的我也見識到資訊領域的面試是什麼型態。

目前狀況是，在這也做了一年多了，公司有點轉型變成接案性質了，也因為這樣讓我一下子把自己的 Vue 壓縮起來學習的很快，可是第一個案子之後深刻感受到，團隊有一個會技術的 PM 是很重要的，因為現在的 PM 就是被客戶牽著鼻子走，外面到處開支票再回來弄自己的工程師（早知道這麼輕鬆我也去應 PM 講講幹話拍拍客戶的頭就好😗），搞得現在的狀況是，同時接了三個案子，交期重疊在一起，而且什麼屁規格都沒確定唯一壓死的是 deadline !? 沒有規格，沒有需求，你就有自信先說什麼時候會好？於是就造就了 後端開 api → 前端串 api → 客戶邊改需求 → 後端改資料庫 → 前端重串(幹)，既然這樣那就希望這個 PM 有能力再去拍拍客戶說這個專案絕對會 delay 囉🤷‍

接案性質的工作讓我覺得可以學到的東西很廣，因應不同的需求可以接觸到很多沒有做過的功能，但也僅限於堪用，做完功能沒問題要趕著繼續往下走其他頁面，所以有過外包經驗後我現在的方向則是朝有產品包裝的公司為目標，可以瞭解如何維護產品及提升產品價值的 work flow，包含以前完全沒時間用到的測試、git flow、自動化等等，不同於前者可以深入的方向鑽研。

首先先介紹自己的能力 (以求職網上常見的需求列出)：

  • HTML5, CSS3
  • RWD
  • 前端任一框架: Vue
  • 熟悉 ES6
  • build system: gulp, webpack
  • 解決跨瀏覽器問題
  • 串接 api 經驗
  • 熟悉 git 版控

再來是我個人覺得可以寫上去的一些經驗

  • 自己玩過前端測試：e2e - cypress, unit - jest
  • PWA 作品
  • 有後端經驗(Ruby on Rails)

FunNow

新創公司，產品定位有點像 EZtable 但功能更強大預約 app，服務的層面很廣，從飲食到住宿甚至球賽都有，因為剛獲創投資金正在廣納人才。面試當天先寫了一份忘了是性向測驗還是智力測驗的題目，就是很多形狀或圖形要判斷空格內應該填入哪個圖形的考題，總共 15 題只寫了 11 題瞬間覺得原來我智能不足 … 技術部份倒是沒有考試，面試的前端說太忙了沒時間出，之後如果錄取可能要先 full-stack 等人手補足才會好好的整頓一下負責工作，他們後端目前是用 php 但預計轉向 golang。面談期間其實蠻輕鬆的，大概 demo 一下作品跟自己的部落格、網站，然後聊聊前端在公司內的定位(因為主要產品是 app)，也有提到說不是資訊科不懂一些 CS 可能會比較辛苦等等。最後突然拋出説不然來考一下試好了 !?(馬上荒髒😱)，一題是從陣列隨機拋三個不重複的數出來，第二個是去年的夢靨：費氏數列(真的很費事)，這部分就跟他帶過一下自己有寫過但真的不熟 … 總之，人手看起來是很缺，但也因為產品要衝刺我覺得新人也會慎選不會太隨便，所以不是很有把握。

結果：感謝函

Stockfeel 股感

這間是在數字網看到的，展示的工作環境看起來很不錯就投了，後來才知道公司已經搬家，現在完全不一樣 … (詐騙)。同時面試一個人資兩個前端，大概展示一下作品然後自我介紹之後那個人資就批哩啪拉的一直講他們公司在幹嘛做什麼的，也不管人家受不受得了，我大概第三句的斷線了😐，但其實斷線的主要原因是聽到要跟各大金融業合作，眾所皆知金融業系統老舊還有那個人見人怕的 Internet Explorer，興致垂直驟減 (現任公司要支援挨伊就很讓人暴走了還跟你在那邊倒退走 …)。總結這家就是當練習，但應該不需要 Senior 太多的特質，如果去年來的話搭個作品就可以上班了的感覺。

結果：無聲卡

inline 樂排

這家算是我排期待的一間公司，因為他有 升 降 桌 ，一個非常吸引人的特點啊！產品跟 FunNow 一樣主要定位在 app 上，兩者有點像上下游的關係，inline 主要幫餐飲業簡化定位及帶位流程，進一步減少客戶等待時間。第一次面試對方用 phone interview 所以沒有進到公司了解環境的機會，前端未來也會需要在網頁上跟 app 同步功能，讓中小型的店家不需要 ipad 也可以方便使用，面談後對方會給回家功課來評估是不是要進一步二面。收到的題目是：用 Vue + firebase 做一個即時聊天網頁(小竊笑了一下)，但這次不能馬虎，其實也有想過想多加幾個優化的 feature 進去，但怕讓對方等太久還是上了基本功能後叫交卷了，另一方面是公司專案最近也在軋時間。於是很順利地進到了第三階段的面試，進到公司可以感受到一群人凝聚的那種溫馨感，第一部分先跟兩位前端前輩面試大概聊一下對 Vue 的理解如何，做專案有遇到什麼困難等等，整個過程讓人感覺不會太嚴肅有壓力，問的一些問題也讓我直接開電腦 google 感覺是在試探找答案的能力，有提到目前他們兩位都還要負責 cover 後端的部分，這一點後面第二部分也有跟創辦人談了一下，畢竟現在職缺是找「前端工程師」，所以也不會太強求要先有這方便的技能，可以進來再慢慢學，然後還要兩人一組輪班當客服幫忙解決業者在使用上的問題。至於還是那一個重點問題：前端在 app 為主的產品公司扮演什麼樣的角色？創辦人才提到他們想要在網頁上也能讓小型企業可以不用倚靠 iPad 一樣使用我們的服務，更進一步的想要做到讓使用者直接對餐廳訂位的功能，這方面就需要前端來實作，所以整體下來，從面試流程到工作目標我覺得 inline 給我很完整的體驗與資訊，很期待能夠進來學到更多的東西！

結果：口頭 offer get(尚未收到正式信)

東南旅行社

目前為止這家最令我進退兩難，進的點是團隊很大，而且設計部門有程式底，自己會切版，不會天馬行空的想一堆跟作業時間成反比的東西 … 要就自己切！除了維護旅行社官網(React)以外，基本上有活動頁面都是可以自由發揮的(不限定 React)，也有保障年薪 13 個月、專案獎金、旅遊補助等等，福利面不錯，而且工程部使用 git flow 開發，聽起來是有很嚴謹的作業團隊；而退的點是，辦公室空間很小很矮 … 沒錯很矮，大概我手向上伸直就快把天花板打掉了，整個空間壓力很大，而且人多平面空間也壅擠，再來是有硬性服裝規定：襯衫、西裝褲、皮鞋，這樣導致上班的心情就很差，不想每天自己忍受這樣的環境，我應該大概也許可能是不會想來這裡吧。

結果：無聲卡

目前一週內面試了四家，很巧妙的一天排兩個只用掉兩個下午的假✌️，但其實礙於現在的專案進度會審慎安排接下來的面試時間，如果風調雨順的話我的優先順序應該會是

inline > FunNow > else …

再來有關所謂 Senior 的資格，面試期間大家都是從專案作品上了解，實際有參與網站製作的經驗其實應該就符合大多數公司的門檻了，除了 FunNow 有談到關於 CS 的東西之外其他都還算順遂，也可能只是還沒遇到非常嚴格的面試考試，如果真的考應該會直接慢走不送吧，因為我自己覺得自學的缺點是對於程式基底的知識非常貧缺，像問什麼 stack 啊 queue 啊、prototype 會不會、MVC 架構、OOP 概念等等，我也許會用，也許有用過，但我真的不知道他的理論跟名字 … 面試完 FunNow 後我也對 golang 起了一點興趣，目前有時間就會研究一下用它來寫寫看後端 server，雖說前後 js 一家親是很好，可是現在 Nodejs 連爸爸都不要他了 😦

後記：09/16

上次的面試後還是繼續投發履歷，期間也接到了兩個面試

Pic Collage

沒錯就是一家拼貼 app 公司，跟上次 inline 一樣，很好奇 app 公司對於前端的需求會是什麼，也很巧他認識我們 AC 校長 Bernard 就有稍微來往一些寒暄的信，中間間隔一段時間沒下文我一度以為被摸摸頭了，但後來有收到好像是技術長的面試邀請。第一次採視訊面試，整體面試過程感覺氣氛是很輕鬆的，還會問你平常下班會不會打電動打什麼之類（齉你卸下防備），然後不免俗的自我介紹、離職（求職）動機等等，公司目前沒有所謂專職前端的部分，算是近期才開始因為 facebook 相關產品的關係，要打入 web product 的市場，像是拼貼卡片、結合 3D 模組的節日賀卡等等，說是不排斥其他框架的應用但是 React 是最好，至少他們認為有 FB 背書才能保障產品走得遠，後來有一個 live coding

寫一個可以套用上方需求的 function，但他們用的 live editor 其實也沒辦法做驗證 …(算是賺到嗎)，只是純粹可以看到我正在打的 function，所以就用一邊解釋一邊修正的方法來進行面試，但不得不說 live coding 是我最討厭的一環，有人盯著你是要怎麼思考，壓力山大的，所以他也是邊提點讓我邊校正，最後提到如果順利進到下一步還有另一位技術人員再約一次 con call，但我看我的表現應該是渺茫了😓。

補充：除了這題還有問 CSS margin collapse 的題目，應該不難，只是我聽他講才這個叫 Box model …常用但未必知道他叫什麼啊～

創順科技

這段面試包含很多個人情緒，參考參考就好

必須說這是我至今面試遇到極差沒有之一的公司，這份是從 104 被他們人資邀約才考慮面看看，不然地點在內科，而且離捷運站也有一段路就覺得上班好麻煩 …，從他們的徵才簡介上根本看不出公司是在幹嘛的，剛好就排在病假那天抽個時間去會一會，但我後悔了，是非常後悔。人資領進門後，劈頭進來的就是一個技術人士，為什麼說是「技術人士」呢，因為他講話含糊不清，咬字全連在一起還有氣無力的，根本聽沒有他在公三小，一坐下來也不客氣的說，我看你不是相關科系畢業的，為什麼會想要寫網頁，就大概交代一下來龍去脈，下面問題像是

問：你 landing page 都怎麼切？
答：就是先把設計稿的版面切分看要怎麼畫架構，包含考慮 RWD 等等的因素，再設定變數和 template
問：不是，我是問你你知道要做 landing page 後會先做什麼？
（我他媽不是剛講完嗎？）

問：你自認為 html, css, javascript 比較擅長哪一塊？
答：應該是 CSS (JS 因為基礎不是很好所以不敢說擅長，但切這麼多版了就算不到精通也還說的上熟練吧)
問：那你知道 class 怎樣比較快嗎？
答：哦你是問權重嗎？就是先 ID 再來 bulabula(跳過) 會有優先順序，如果是要優化速度的話就盡量減少 css 的巢狀層級或避免使用 * 這種無差別選擇器
問：不是（唉），我是問你寫哪種 class 他速度會比其他人快
（我真他媽不是聽不懂你在講什麼就是我才疏學淺第一次聽到有寫某個 class 就能跑得比其他 class 快的說法）
答：你說的速度是指哪方面的速度？
問：現在是你要回答我還是我要回答你？
（到這邊我就理智斷線了，下面回答一切從簡，反正在我這他已經被刷掉了）

Vue 底層怎麼運作？ 不知道
Scss 它的原理是什麼？ 不清楚
哦有學 React，那你知道 React 他重點是什麼？ 畫面，也就是 View
那他的實作原理是什麼？ 不知道
用過 webpack 那他作用是什麼？ 幫我們處理壓縮打包整合編譯源碼的作業
不是，你有讀過他的源碼嗎？它實際上在做什麼怎麼做？ 不知道

等等，因為歸藍趴火也沒什麼再記他講什麼，更別說他的咬字發音聽了也很痛苦，後面考了一個我覺得工程師都不應該遇到的手寫 code，最最最簡單，你也開個 chrome devtool 來，黑筆白字？我是來進京趕考的嗎，我真真切切的不理解那些想靠紙筆寫扣來面試的公司期望是什麼？除非我進公司就是一個紙上工程師，全部寫完再讓其他「工程師」去 key，不用 debug。總之這個人整個態度就是姿態很高，我是看了徵才條件自認都可以做到公司所要求的才來試試看，但你一直要跟我提什麼底層基礎原理，我大膽的認為大部分在使用框架的人不會去接觸到他裏層的東西，因為光表面我們就有無止境的東西要看要學，確實了解它的原理是很重要這不否認，可是你今天張貼公告要的是一個會使用框架來解決問題的人，我卻沒看到你怎麼評估我解決問題的方式跟邏輯，還是其實這一切只是想打壓你所謂「非相關科系畢業」的應試者？那你成功了，後來人資進來講公司福利的部分，他們交接的空擋我也查了一下這是一間博弈公司，專做什麼三小彩票系統的，公司福利再好，第一印象遇到這種嘴臉的我連想都不敢想要怎麼跟他合作，總而言之，祝貴公司鴻圖大展。

最後總結一下，我覺得面試這檔事，雙方都應該彼此尊重才是，因為難保今天的應試者能力遠超過你們公司所需要的，總是要表現出想要留住人才的誠意吧，就跟我們去面公司一樣，都是一體兩面的，不只是公司在挑人，人也在挑公司，不要搞得好像我是資方我招人就最大的態度，要搞清楚現在是你需要人來幫你解決問題；而像我這種只會一點皮毛的當然不敢太強勢，只能說我的策略是手先搆到高處在讓身體往上爬，讓公司來驅動自己成長，並不是自己不會進修啦，只是我自己是要在實戰場合才能知道自己其實有幾兩重，不夠就趕快補起來的那種人🧐（不喜歡拖累團隊），之後也不知道會不會繼續安排面試，就這樣平平穩穩的進 inline 好好磨個幾年功，看能不能改一改自己這種能力上的缺陷。
